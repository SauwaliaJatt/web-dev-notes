HTML Elements:-

The HTML element is everything from the start tag to the end tag:
<tagname> Content goes here...</tagname>

Paragraphs:-

The HTML <p> element defines a paragraph.
A paragraph always starts on a new line, and browsers automatically add some white space (a margin) before and after a paragraph.
Usually a block ot text.

Headings:-

HTML headings are titles or subtitles that you want to display on a webpage.
HTML headings are defined with the <h1> to <h6> tags.
<h1> defines the most important heading. <h6> defines the least important heading.

Lists:-

HTML lists allow web developers to group a set of related items in lists.

Unordered Lists:-

The HTML <ul> tag defines an unordered (bulleted) list. Each list item starts with the <li> tag.
The list items will be marked with bullets (small black circles) by default.
The CSS list-style-type property is used to define the style of the list item marker. It can have one of the following values:

  • disc (default)
  • circle
  • square
  • none

Ordered Lists:-

An ordered list starts with the <ol> tag. Each list item starts with the <li> tag.
The list items will be marked with numbers by default.

Type Attribute -- The type attribute of the <ol> tag, defines the type of the list item marker
values = 1,A,a,I,i

  1. type = "1" -- Numbers(default)
  2. type = "A" -- Uppercase letters
  3. type = "a" -- Lowercase letters
  4. type = "I" -- Uppercase Roman numbers
  5. type = "i" -- Lowercase Roman numbers

Control List Counting -- If you want to start counting from a specified number, you can use the start attribute:

Note: A list item (<li>) can contain a new list, and other HTML elements, like images and links, etc.

Description Lists:-

A description list is a list of terms, with a description of each term.
Use the HTML <dl> element to define a description list
Use the HTML <dt> element to define the description term
Use the HTML <dd> element to describe the term in a description list

Attributes:-

HTML attributes provide additional information about HTML elements.
All HTML elements can have attributes.
Attributes usually come in name/value pairs like: name="value".
Attributes are always specified in the start tag.
e.g - href, src, width, heigth e.t.c.

Anchor Element:-

The <a> tag defines a hyperlink, which is used to link from one page to another(used to add links to your page).
The href attribute specifies the URL of the page the link goes to.
<a href="link/path"> text/image e.t.c </a>

Image tag:-

The HTML <img> tag is used to embed an image in a web page.
The tag has two required attributes:
src - Specifies the path/link to the image.
alt - Specifies an alternate text for the image if the browser cannot find the image.

Sementics Markup:-

A semantic element clearly describes its meaning to both the browser and the developer.
Means we can find what a tag is used for by its name. Make code more meaningfull and SEO friendly.
Examples of non-semantic elements: <div> and <span> - Tells nothing about its content.
Examples of semantic elements:
<header> - Defines the introductory section or header of a page or section. Used only one time.
<footer> - Represents the footer section, typically containing metadata or links. Used only one time.
<nav> - Specifies a navigation menu for links within the site.
<main> - Represents the central content of a webpage. Used only one time.
<section> - Defines thematic grouping of content, often with a heading.
<article> - Represents self-contained content like blog posts or news articles.
<aside> - Specifies complementary or sidebar content.
<figure> - Groups media content like images with captions.
<figcaption> - Provides a caption or description for the <figure> element.
<mark> - Highlights text to indicate relevance or importance.
<time> - Represents a specific point or duration of time.
<details> - Creates a collapsible disclosure widget for additional information.
<summary> - Defines a summary or label for the <details> element.
Clearly defines its content.

HTML Entities:-

Reserved characters in HTML must be replaced with entities.
An HTML entity is a piece of text ("string") that begins with an ampersand (&) and ends with a semicolon (;).
Entity names(we prefer name) or entity numbers can be used to display reserved HTML characters.
like &entity_name; OR &#entity_number;

Forms:-

An HTML form is used to collect user input. The user input is most often sent to a server for processing.
The HTML <form> element is used to create an HTML form for user input.

Action Attribute:


The action attribute defines the action to be performed when the form is submitted.
Usually, the form data is sent to a file on the server when the user clicks on the submit button.
<form action="/action_page.php"> - In this example , the form data is sent to a file called "action_page.php".
Note: If the action attribute is omitted, the action is set to the current page.

<input> Element:

One of the most used form elements is the <input> element.
The <input> element can be displayed in several ways, depending on the type attribute.

Type attributes types
Type attribute value Working
text default value of the type attribute is "text".defines a single-line text input field.
password The characters in a password field are masked
email The type="email" is used for input fields that should contain an e-mail address.
Depending on browser support, the e-mail address can be automatically validated when submitted.
submit defines a button for submitting form data to a form-handler.
The form-handler is typically a server page with a script for processing input data.
The form-handler is specified in the form's action attribute.
radio Radio buttons let a user select ONLY ONE of a limited number of choices
•We have to use the same name attribute value for multiple radio buttons to group them together, allowing only one to be selected.
•We must use the value attribute according to the option, so that when the form data is sent to the server,
the selected option can be identified.
checkbox Checkboxes let a user select ZERO or MORE options of a limited number of choices.
•We can use "checked" attribute to tick by default
•We have to use the same name attribute value for multiple checkboxes if they represent a group of related options,
allowing multiple options to be selected.
•We must use the value attribute according to each option so that when the form data is sent to the server,
all selected options can be identified.
number type="number" defines a numeric input field.
You can also set restrictions on what numbers are accepted.
date is used for input fields that should contain a date.
Depending on browser support, a date picker can show up in the input field.
You can also use the min and max attributes to add restrictions to dates.
file defines a file-select field and a "Browse" button for file uploads.
tel used for input fields that should contain a telephone number.
range defines a control for entering a number whose exact value is not important (like a slider control).
•Default range is 0 to 100.
However, you can set restrictions on what numbers are accepted with the "min", "max", and "step" attributes. • we can use value = "x" to select a x value by default.

placeholder attribute:-

The input placeholder attribute specifies a short hint that describes the expected value.
The short hint is displayed in the input field before the user enters a value.
The placeholder attribute works with: text, search, url, number, tel, email, and password.
e.g. - <input type = "password" placeholder = "Enter password here"> -

<select> element:-

The <select> element defines a drop-down list.
The <option> element defines an option that can be selected.
By default, the first item in the drop-down list is selected.
To define a pre-selected option, add the "selected" attribute to the option.
e.g of select element:- < select name="profession" id = "profession">
< option > Select your option </option>
< option value= "student"> Student</option>
< option value= "dev" selected>Developer</option>
</select>
Use the size = "n" attribute to specify the number of visible values at once.

<textarea> element:-

The <textarea> element defines a multi-line input field (a text area).
The "rows" attribute specifies the visible number of lines in a text area.
The "cols" attribute specifies the visible width of a text area.
You can also define the size of the text area by using CSS.

name attribute:-

The name attribute in an <input> element defines the key used to send the input's value as a key-value pair in form submissions.
Each input element must have a unique name unless they are part of the same group (e.g., radio buttons sharing the same name).
